## Overview

Led the design and development of an AI-powered sales chat agent that allows sales representatives to query both structured and unstructured data using natural language.

The system integrates Text-to-SQL, retrieval-augmented generation (RAG), entity extraction, and knowledge graphs to provide answers ranging from KPI-driven analytics to document-based insights. It was deployed at scale across a large pharmaceutical organization, serving 50k+ users.

## Problem

- Sales reps relied on multiple disconnected tools for analytics, documents, and insights.
- Querying structured data required technical expertise (SQL, dashboards).
- Unstructured knowledge (documents, reports) was difficult to search effectively.
- No unified interface existed to combine structured and unstructured insights.
- Existing systems were slow, fragmented, and required manual effort to extract insights.

## Constraints

- Needed to support multiple query types: structured, unstructured, and hybrid.
- High scale (50k+ users) with strict latency requirements.
- Sensitive enterprise data requiring strong validation and guardrails.
- Cross-team dependencies across data, platform, and frontend teams.
- Aggressive timelines for delivering a production-ready v1.

## Approach

Designed the system as a **multi-agent architecture**, where specialized agents handle different query types and are orchestrated through a central routing layer.

Core capabilities included:
- Text-to-SQL for structured analytics queries
- RAG pipelines for document-based question answering
- Hybrid pipelines combining database and document retrieval
- Entity extraction and knowledge graph validation for accuracy and safety

Prioritized:
- Clear separation of responsibilities between agents
- Strong validation layers before executing queries
- Fast routing and response generation for interactive usage

## System Design

The system routes user queries to specialized pipelines based on intent.

**Structured queries:**
User queries are parsed for intent and entities, mapped to predefined KPIs, and converted into SQL using an LLM. Outputs are validated and transformed into structured responses used for downstream data visualization.

**Unstructured queries:**
Documents are retrieved using semantic search and passed into an LLM along with contextual signals (entities, knowledge graph data) to generate answers.

**Hybrid queries:**
Structured and unstructured pipelines run in parallel, and outputs are combined into a final response with both data and narrative summaries.

A routing layer determines which pipeline to invoke, enabling a unified user experience across all query types.

## Key Decisions

### Build a unified multi-agent system instead of separate tools

Instead of building independent agents for each use case, a centralized architecture with shared infrastructure (routing, prompts, evaluation, security) was created.

This reduced duplication, improved consistency, and made it easier to scale and maintain.

### Introduce a routing layer for intent-based orchestration

Queries vary significantly in structure and requirements. A routing layer allowed the system to dynamically select the appropriate pipeline (structured, unstructured, hybrid).

This improved both accuracy and efficiency by avoiding unnecessary computation.

### Add validation layers before executing model outputs

LLM-generated outputs (e.g., SQL) were validated using structured schemas and knowledge graph checks before execution.

This reduced risk of incorrect queries, improved safety, and ensured consistency with enterprise data definitions.

### Prioritize latency and system responsiveness

Optimized the system for interactive use by:
- Reducing model and pipeline overhead
- Introducing caching and efficient data access patterns
- Minimizing unnecessary downstream calls

This resulted in an ~82% reduction in response latency.

## Results & Impact

- **Scale:** 50k+ active users across the organization
- **Latency:** ~82% reduction in response time
- **Adoption:** Increased usage of analytics and document exploration tools
- **Productivity:** Reduced reliance on manual querying and fragmented systems

The system unified multiple data access patterns into a single interface, significantly improving how sales teams interact with data.

## Tradeoffs

- Multi-agent architecture increased system complexity and coordination overhead.
- LLM outputs required extensive validation and monitoring to ensure reliability.
- Evaluation of system quality remained challenging due to subjective responses.
- Hybrid queries introduced additional latency due to parallel processing.

## Learnings

- Multi-agent systems are effective for complex enterprise use cases but require strong orchestration.
- Validation layers are critical when LLM outputs drive downstream execution.
- Latency optimization is essential for user adoption in interactive systems.
- Evaluation frameworks for LLM systems require both automated and human-in-the-loop approaches.

## Future Work

- Improve intent routing with better contextual understanding.
- Introduce deeper personalization for user-specific insights.
- Expand evaluation frameworks with automated quality metrics.
- Enhance explainability of responses to improve user trust.
