## Overview

The Next Best Message system recommends the most effective message a sales rep should send to a given account. It uses historical interaction data, account context, and message attributes to rank candidate messages by likelihood of engagement.

The goal was to reduce decision friction for sales reps while improving consistency and effectiveness of outreach across accounts.

## Problem

- Sales reps manually select messages with little data-driven guidance.
- Message effectiveness varies significantly across accounts and contexts.
- Reps spend time experimenting instead of executing high-probability actions.
- No system existed to learn from past message outcomes and improve over time.

## Constraints

- Sparse and noisy feedback (limited explicit success signals).
- High variability across accounts, reps, and message types.
- Needed low-latency recommendations within existing workflows.
- Small team with limited time to build and productionize (~3–4 months).

## Approach

Framed the problem as a ranking task: given an account and its context, rank a set of candidate messages by likelihood of success.

Built a hybrid recommendation system combining:
- Behavioral signals from past interactions
- Contextual features (account attributes, timing, recency)
- Lightweight ranking models to enable fast iteration and deployment

Prioritized:
- A reliable interaction data pipeline
- Feature engineering over model complexity
- Fast iteration cycles with measurable feedback loops

## System Design

The system follows an offline training and near real-time inference pattern.

Interaction data is collected from sales rep activity and transformed into structured features capturing account context, message characteristics, and temporal signals. A ranking model is trained offline and periodically updated.

At inference time, the system scores a set of candidate messages for a given account and returns a ranked list via an API used in the sales workflow.

## Key Decisions

### Frame the problem as ranking instead of classification

Ranking candidate messages directly aligned with how the system is used in practice—selecting the best option from a set—rather than predicting independent success probabilities.

This also made the system more robust to sparse labels and easier to evaluate in relative terms.

### Use a hybrid recommendation approach

Pure collaborative filtering was insufficient due to sparse interactions, and content-based methods alone lacked personalization.

Combining behavioral signals with contextual and message-level features improved generalization across accounts and reduced cold start impact.

### Start with simple models and batch updates

Instead of building a complex real-time learning system, the initial version focused on batch-trained models with lightweight scoring.

This reduced system complexity, enabled faster iteration, and made debugging significantly easier during early deployment.

## Results & Impact

- **Engagement:** Improved interaction rates on recommended messages (directionally positive)
- **Efficiency:** Reduced time spent selecting messages for outreach
- **Consistency:** Standardized messaging quality across reps

The system introduced a feedback loop where rep interactions continuously improved future recommendations, turning a previously manual process into a learning system.

## Tradeoffs

- Sparse and indirect feedback limited early model accuracy.
- Batch updates simplified the system but reduced responsiveness to immediate behavior changes.
- Cold start scenarios for new accounts and reps required fallback heuristics.
- Limited explainability initially made it harder to build user trust.

## Learnings

- Ranking formulations better match real-world decision systems than classification.
- Hybrid recommenders are effective defaults in sparse enterprise environments.
- Data quality and feature design matter more than model complexity early on.
- Offline metrics alone are insufficient—proxy engagement metrics are critical.

## Future Work

- Introduce contextual bandits to balance exploration and exploitation.
- Improve cold start handling using knowledge-based priors.
- Move toward real-time feature updates and online learning.
- Add explainability to improve rep trust and adoption.
