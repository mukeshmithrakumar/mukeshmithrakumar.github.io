### Introduction

Concurrent processing is a computing model in which multiple processors execute instructions simultaneously for better performance. Concurrent means something that happens at the same time as something else. Concurrent processing is sometimes said to be synonymous with parallel processing.

By the end of this module, you will be able to:
- Write the differences between concurrency and parallelism
- Write the differences between threads and processes
- Write the advantages and disadvantages of concurrent processing
- Define kernel threads and user threads
- Write the advantages of kernel threads and user threads
- Define read - write and write-write conflicts
- Write pseudocode examples of race conditions
- Given pseudocodes with race conditions write the possible outputs of the code for multi-threaded execution
- Write mathematical proofs of mutual exclusion, progress, and bounded waiting for different critical section solution algorithms
- Define semaphores
- Write pseudocode using semaphores to solve different process synchronization problems such as dining philosophers, readers and writers, and producer consumer
